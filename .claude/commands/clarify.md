# /clarify — grill me for clarity

Read the referenced requirement/request and ask clarifying questions

| Arguments | Scope |
|---|---|
| `next` | peform this skill on the next unfinished FEATURE in the Backlog.md file  |


## How to work each item

Follow this sequence:

-* **Identify** - verify with user that you have identified the right target item(s).
-* **Understand** - Consider the goal of the request, especially the business goal(s). If goal is unclear, ask user. 
-* **Question** - consider whether or not the request is unwise, inappropriate, or perhaps redundant. If so, express your conclusion and reasoning to user and ask for next steps. 
-* **Consider alternatives** — is there a better way to achieve the goal? If there are strong or typical alternatives that should be considered, present them to user and ask for next steps. 
-* **Consider edge cases** — what edge cases should be considered that have not been specified? Ask user about those and how to handle them.  
-* **Update** — Update the identified target item to reflect these inputs so that the request can better prompt AI for later implementation.
